# ODB SADCP 科普研究重現計畫

## Current Baseline

- 已完成 ODB SADCP API 的公開資料繪圖驗證。
- 已完成兩條技術路線的基礎圖像：
  - [第一版 SADCP 2D 海流圖](/Users/cywhale/proj/apigate/examples/kurshio_current/README.md)
  - [第二版 SADCP + GEBCO 疊圖](/Users/cywhale/proj/apigate/examples/kurshio_current_v2/README.md)
- 已知可用能力：
  - 0.25 度海流色階散點
  - 25-35 m 近表層箭頭
  - 20-300 m 深層平均背景流速
  - `mode=17` 與 `mode=18` 季風比較
  - GEBCO 半透明地形背景
- 已知不可做：
  - 私有資料庫的 0.1 度補格
  - 原始航次等級的事件重現
  - 僅靠 ODB SADCP 判定溫鹽水團結構

## Goal of Next Iteration

選出一個可由 ODB SADCP 公開 API 支撐的科學研究案例，完成「具科學意義的單一圖組或單一分析問題」之實作規劃，而不是整篇論文的全量重現。

## Constraints

- 只能使用公開 ODB API 與可公開取得的補充資料。
- SADCP 為 0.25 度格點，適合結構與氣候平均，不適合細尺度事件。
- 不能宣稱重現原論文的私有或高解析產品。
- 若全文無法取得，不可把該論文選為主攻案例。

## Candidate Paths

## Path A: 黑潮分支流與鰻苗輸送

- 摘要: 以 ODB SADCP 建立呂宋海峽到臺灣海峽的 KBC 流場類比，回答「哪些季節或背景流況下較容易出現黑潮向西北侵入南海與海峽的流場型態」。
- 預期收益: 科學問題清楚，且最容易做成對外可解釋的圖組。
- 科學風險: 無法直接重做 ENSO 年門檻與漁獲統計，只能做流場 proxy。
- 成本: 中。
- 阻塞依賴: 需要明確定義 branch-current proxy。

## Path B: SAE 深海飛碟

- 摘要: 以 ODB SADCP 嘗試做南海北部次表層渦旋流場類比。
- 預期收益: 故事性強。
- 科學風險: 高。沒有溫鹽資料就不能說已辨識 SAE。
- 成本: 中高。
- 阻塞依賴: 需要全文與更多水文資料。

## Path C: 黑潮大蛇行與熱浪

- 摘要: 將黑潮路徑異常與氣候效應做科普化說明。
- 預期收益: 故事很強。
- 科學風險: 極高。核心機制不是 ODB SADCP 在臺灣區域能回答的。
- 成本: 高。
- 阻塞依賴: 需要更多海氣資料與事件期資料。

## Path D: 東臺灣黑潮平均結構

- 摘要: 以公開 ODB 流場重現黑潮在臺灣東側的平均結構與變動。
- 預期收益: 最穩、最容易驗證。
- 科學風險: 低。
- 成本: 低。
- 阻塞依賴: 敘事較弱，對外吸引力不如 KBC。

## Recommended Path

選擇 Path A，也就是 [Han et al. 2021](https://doi.org/10.3390/jmse9121465) 相關的 KBC 與鰻苗輸送案例。

理由：

- 能把 ODB 公開 API 的強項直接轉成一個具科學意義的圖組。
- 即使不碰事件年資料，仍可回答「哪類背景流場較支持黑潮分支侵入」。
- 和前面已完成的 Kuroshio 圖像技術路線相容，開發成本最低。
- 對外敘事清楚，較容易被非專業讀者理解。

## Phase Plan

## Phase 1: 文獻對位與圖組定義

- 目標: 鎖定要對應 Han et al. 2021 的哪一個科學問題與哪一組類比圖。
- 輸出:
  - 主攻圖組定義
  - branch-current proxy 定義
  - 研究區域與斷面設定
- 驗證:
  - 必須能用一句話說明圖回答的問題
  - 必須能明確區分重現與類比範圍

## Phase 2: ODB API 分析設計

- 目標: 明確化 API 參數、資料流程與圖面布局。
- 輸出:
  - 深層背景流速圖
  - 近表層向量圖
  - 月平均或季風比較圖
  - 一個 branch-current proxy 圖或表
- 驗證:
  - 參數不含私有資料依賴
  - 所有圖都能用已存在的 ODB 繪圖框架完成

## Phase 3: 驗證與敘事包裝

- 目標: 形成可對外說明的科學圖像與文字框架。
- 輸出:
  - 圖說與限制聲明
  - 與論文結論對照的驗證清單
  - 若主攻圖失敗時的備案切換條件
- 驗證:
  - 不過度宣稱
  - 讀者能理解「這是研究類比，不是事件重演」

## Validation Criteria

- 數值檢查:
  - branch-current proxy 的定義可重算
  - API 回傳欄位與深度設定一致
- 視覺檢查:
  - 能清楚看出 Kuroshio 主軸與向西北入侵趨勢
  - 季風或月份對照圖之間有可讀差異
- 科學檢查:
  - 圖像回答的是 KBC 入侵條件，不只是海流美圖
  - 所有強主張都能回溯到文獻
- 再現性檢查:
  - 參數與步驟可以被另一個實作 agent 直接執行

## Open Questions

- branch-current proxy 最適合用固定斷面上的西向流量比例，還是用目標區域內西北向速度面積比例來表示。
- 主攻圖組是用月平均序列比較較好，還是先用 `mode=17` / `mode=18` 的季風雙圖較好。
- 是否需要把 GEBCO 地形作為主圖背景，或僅作輔助版本。
